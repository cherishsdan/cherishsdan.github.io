Module

  • javascript
  • module

关于模块化：

  • 可重用
  • 封装了变量和function，和全局的namaspace不接触，松耦合
  • 只暴露可用public的方法，其它私有方法全部隐藏

1、基本的模块化

对象写法

`
var Module = {
m1: function(){},
m2: function(){}
}

`
减少 Global 上的变量数目,起到了一点点的命名空间作用
同样缺点明显: 会暴露所有模块成员，内部状态可以被外部改写

匿名闭包 ：IIFE 模式

`
var Module = (function(){
var _private = “safe”;
var m1 = function(){
/ code /
}

return {
    m1: m1
}

})()

Module.m1();
Module._private; // undefined
`
可以很好的保护私有变量，通过return来设置公开的方法
动态添加方法的时候比较麻烦，且无法修改内部私有变量.

##引用全局变量
`
var Module = (function($){
var _$body = $(“body”); // can use jQuery
var m1 = function(){
/ code /
}

return {
    m1: m1
}

})(jQuery)

Module.m1();
`
全局变量当成一个参数传入到匿名函数然后使用.

`
var Module = (function () {
var m = {},
_private = “”;

function privateM1(data) {
     /* code */  
}

m.attr = _private;
m.method = function (data) {
    privateM1(data);
};

return m;

} ());
`
全局变量Module带有2个可访问的属性Module.method和Module.，除此之外，其它代码都在匿名函数的闭包里保持着私有状态。

module 扩展

将一个功能分离成多个文件
var Module = (function (m) { m.method = function () { /* code */ }; return m; } (Module));

松耦合扩展

解决顺序问题
var Module = (function (m) { m.method = function () { /* code */ }; return m; } (Module || {}));
首个被引入的文件会创建Module命名空间（就是个JavaScript对象），而且所有使用这种结构的后续文件都只是重用此现有实例。

紧耦合扩展

紧耦合扩展限制了加载顺序，但是提供了我们重载的机会
`
var Module = (function (m) {
var oldMethod = m.AddMethod;

m.AddMethod = function () {
    /* code 
    可通过oldMethod调用旧的方法
    */
};

return m;

} (Module));

`
